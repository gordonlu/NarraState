# 真相变体设计

`CaseTemplate` 保存地点、人物、事实、证据、陈述、场景和结案规则等共享内容；每个 `SolutionVariant` 用显式覆盖定义责任人、动机、行为链、证据链、披露图和结局。编译器把模板与一个变体合并成可直接运行的完整 `CompiledCase`，运行时不再做覆盖合并。

有意义的变体不能只替换责任人姓名。校验器会比较责任人、动机、关键行为、核心证据、关键矛盾和结案路径；差异不足返回稳定错误码 `VARIANT_INSUFFICIENT_DIVERGENCE`。

新游戏支持三种选择：

- `Default` 使用 manifest 的默认变体。
- `Random` 只在已启用且通过编译、校验、模拟的变体中按权重选择。
- `Specific` 用于作者、测试和开发模式。

选择器版本、案件版本和 seed 共同保证可重现性。实例一旦创建，完整编译结果与实例哈希写入 SQLite；Session 只绑定 `instance_id`，以后磁盘案件更新不会改变旧存档。普通玩家 API 不返回 `variant_id`、责任人或隐藏内容。
